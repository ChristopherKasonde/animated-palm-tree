🎯 Complete CSS Grid Guide

What is CSS Grid?
CSS Grid is a two-dimensional layout system that allows you to create complex layouts with rows and columns. Unlike Flexbox (which is one-dimensional), Grid can handle both dimensions simultaneously, making it perfect for page layouts and complex component designs.

11. Grid vs Flexbox

When to Use Which?

🎯 Use Grid For:

  • 2D layouts (rows AND columns)
  • Page layouts and major sections
  • Complex positioning requirements
  • Magazine-style layouts
  • Image galleries
  • Card layouts with varying sizes

🚀 Use Flexbox For:

  • 1D layouts (either rows OR columns)
  • Component-level layouts
  • Navigation bars
  • Centering content
  • Space distribution
  • Alignment of items

13. Advanced Grid Techniques

🔥 Pro Tips:

  • Responsive without media queries: Use repeat(auto-fit, minmax(250px, 1fr))
  • Perfect centering: Use place-items: center (shorthand for align-items + justify-items)
  • Implicit grids: Grid automatically creates rows/columns for extra items
  • fr unit: Represents a fraction of available space (better than percentages)
  • minmax(): Set minimum and maximum track sizes for responsive design
  • Named lines: Use meaningful names instead of numbers for better maintainability
  • Subgrid: Coming soon - inherit parent grid lines in nested grids
/* Advanced Grid Patterns */ /* Holy Grail Layout with Grid */ .page { display: grid; grid-template-columns: 200px 1fr 200px; grid-template-rows: auto 1fr auto; grid-template-areas: "header header header" "nav main aside" "footer footer footer"; min-height: 100vh; } /* Responsive Grid without Media Queries */ .responsive-grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); gap: 2rem; } /* RAM (Repeat, Auto, Minmax) Pattern */ .ram-grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); gap: 1rem; } /* Sidebar Layout that Collapses */ .sidebar-layout { display: grid; grid-template-columns: minmax(250px, 1fr) 3fr; gap: 2rem; } @media (max-width: 768px) { .sidebar-layout { grid-template-columns: 1fr; } } /* Dense Packing Algorithm */ .masonry-style { display: grid; grid-template-columns: repeat(auto-fill, minmax(200px, 1fr)); grid-auto-rows: minmax(100px, auto); grid-auto-flow: row dense; gap: 1rem; } /* Full Bleed Layout */ .full-bleed { display: grid; grid-template-columns: 1fr min(65ch, 90%) 1fr; } .full-bleed > * { grid-column: 2; } .full-width { grid-column: 1 / -1; }

14. Grid Units and Functions

Understanding Grid Units

fr (fractional unit)

/* Takes up fraction of available space */ grid-template-columns: 1fr 2fr 1fr; /* 25% 50% 25% */

minmax()

/* Minimum 200px, maximum 1fr of available space */ grid-template-columns: minmax(200px, 1fr) 2fr;

fit-content()

/* Size to content but never exceed 300px */ grid-template-columns: fit-content(300px) 1fr;

min-content and max-content

/* min-content: smallest size without overflow */ /* max-content: largest size content wants to be */ grid-template-columns: min-content max-content 1fr;

15. Grid Animation and Transitions

/* Smooth transitions for grid properties */ .animated-grid { display: grid; grid-template-columns: repeat(3, 1fr); gap: 1rem; transition: grid-template-columns 0.3s ease; } .animated-grid:hover { grid-template-columns: 2fr 1fr 1fr; } /* Animating grid items */ .grid-item { transition: transform 0.3s ease; } .grid-item:hover { transform: scale(1.05); }

16. Browser Support and Fallbacks

/* Feature detection and fallbacks */ .layout { /* Fallback for older browsers */ display: flex; flex-wrap: wrap; } @supports (display: grid) { .layout { display: grid; grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); gap: 1rem; } } /* IE11 Grid Support (with prefixes) */ .ie-grid { display: -ms-grid; -ms-grid-columns: 1fr 1fr 1fr; -ms-grid-rows: auto; }

19. Performance Considerations

⚡ Performance Tips:

  • Use implicit grids: Let Grid create tracks automatically when possible
  • Avoid deep nesting: Too many nested grids can impact performance
  • Be specific with placement: Explicit placement is faster than auto-placement
  • Use CSS containment: contain: layout style; for better performance
  • Minimize reflows: Avoid changing grid structure during animations

20. Future of CSS Grid

🚀 Coming Features:

  • Subgrid: Inherit parent grid lines (Firefox supported, coming to other browsers)
  • Masonry Layout: True Pinterest-style masonry with grid-template-rows: masonry
  • Container Queries: Responsive grids based on container size, not viewport
  • Aspect Ratio: Better control over grid item proportions
